Appendix C — Per-Material Performance Breakdown (Clean Data, Seed 42)
────────────────────────────────────────────────────────────────────────

Each test split (20% of 305 samples, stratified by material) contains exactly
12 or 13 test samples per material. The RMSE broken down by material gives a
more granular view of where each model is accurate and where it struggles.
Values below are for the Random Forest and MLP on the clean dataset at 501
wavelengths, seed = 42 (single seed, not averaged).

  Material    RF RMSE (nm)    MLP RMSE (nm)    Notes
  ─────────────────────────────────────────────────────────────────────────────
  Si          ~1.6             ~1.2            Smooth dispersion, well-separated spectra
  GaAs        ~2.1             ~1.7            Strong critical-point features near 400 nm
  GaN         ~1.8             ~1.5            Band edge at 365 nm, strong UV absorption
  Ge          ~2.3             ~2.0            High absorption throughout, lower contrast
  InP         ~1.9             ~1.6            Moderate, similar structure to GaAs
  ─────────────────────────────────────────────────────────────────────────────

Note: these per-material values are approximate, derived from the residual analysis
in the diagnostic script (9_diagnostics/diagnostics.py). The exact figures depend on
which 12 test samples fall in the 80/20 split for seed 42. They are included here
for qualitative context rather than as primary results.

GaAs and Ge consistently produce higher error than Si and GaN. For GaAs, the
critical-point spectral features near 400–425 nm create rapidly varying Ψ/Δ
shapes that require more training samples to interpolate accurately. For Ge, the
high and spectrally smooth absorption means that Ψ and Δ vary slowly with
thickness across much of the wavelength range, reducing the effective information
content per wavelength point.

Appendix E — Reproducibility Notes
─────────────────────────────────────

All numerical results in this thesis can be reproduced by running the pipeline
scripts in directory order on any Python 3.9 or later environment with the
following package versions (or compatible equivalents):

  numpy        >= 1.24
  pandas       >= 1.5
  scikit-learn >= 1.2
  matplotlib   >= 3.6

No GPU is required. The entire pipeline from raw n/k data to final diagnostic
plots runs in under one hour on a standard laptop CPU. Random behaviour is fully
controlled by the explicit seed parameters [42, 0, 1, 7, 13] passed to all
stochastic components. The Fresnel simulation and the analytical solver are
deterministic and produce bitwise-identical output across runs.

The noise datasets are generated with a fixed NumPy random state implicitly set
by the order of operations in noise.py. To exactly reproduce the noise files,
run noise.py with no external seed override. The ML results are averaged across
the five explicit seeds, so minor differences in random state during dataset
generation do not affect the reported means or standard deviations.
